📊 Algoritmos de planificación de CPU
FCFS (First Come First Served): Los procesos se ejecutan en el orden en que llegan. Sin preempción. Simple pero puede generar el "efecto convoy" donde procesos cortos esperan tras procesos largos.

📚 Niveles de planificación
Largo plazo
Decide qué procesos se admiten al sistema. Controla el grado de multiprogramación. Se ejecuta con poca frecuencia.
Mediano plazo
Gestiona el intercambio (swapping) entre memoria y disco. Suspend/resume de procesos.
Corto plazo
Selecciona qué proceso en estado Listo obtiene la CPU. Se ejecuta cada pocos milisegundos (planificador de CPU).
🧩 Modelos de gestión de memoria
La memoria se divide en marcos de tamaño fijo. El proceso se divide en páginas del mismo tamaño. No hay fragmentación externa, pero sí interna (última página).
La memoria se divide en segmentos de tamaño variable según las necesidades del proceso (código, datos, stack, heap). Puede haber fragmentación externa.

⚠️ Problemas de concurrencia
Race Condition: Ocurre cuando el resultado de un programa depende del orden de ejecución de múltiples hilos. Ambos hilos leen, modifican y escriben una variable compartida sin coordinación.

val = leer(contador);
val = val + 1; // ← riesgo
escribir(contador, val);
Deadlock (Interbloqueo): Cuatro condiciones de Coffman deben cumplirse simultáneamente: exclusión mutua, retención y espera, sin preempción, y espera circular. El SO puede detectarlo y resolverlo.
Semáforo / Mutex: Mecanismo de sincronización. El semáforo binario (mutex) garantiza que solo un hilo accede a la sección crítica al mismo tiempo mediante las operaciones wait() y signal().
Estado del semáforo Libre (1)
🛡️ Soluciones a problemas de concurrencia
Prevención
Eliminar alguna condición de Coffman. Ej: asignar todos los recursos al inicio (no hay retención y espera).
Algoritmo del Banquero
Verifica si un estado es seguro antes de asignar recursos. Evita deadlocks sin fuerza bruta.
Detección y recuperación
Permite deadlocks, los detecta periódicamente y recupera el sistema mediante preempción o terminación de procesos.
Ignorar (Algoritmo del Avestruz)
Asumir que los deadlocks son tan raros que no vale la pena prevenirlos. Usado en algunos SO de escritorio.

🔧 Herramientas reales para monitoreo
🐧
top / htop (Linux/macOS)
Monitor interactivo en tiempo real. Muestra uso de CPU, memoria, PID, usuarios y permite señales a procesos. htop agrega colores y navegación mejorada.
🪟
Administrador de Tareas (Windows)
Vista gráfica de procesos, rendimiento de CPU/RAM/disco/red, servicios y aplicaciones de inicio. Acceso: Ctrl+Shift+Esc.
💻
ps, vmstat, free, iostat
Comandos POSIX para inspección de procesos (ps aux), memoria virtual (vmstat 1), RAM libre (free -h) y estadísticas de E/S (iostat).
🔬
Valgrind / perf / gprof
Análisis de fugas de memoria (Valgrind Memcheck), perfilado de rendimiento a nivel de syscalls (perf) y profiling de funciones (gprof). Esenciales en desarrollo de sistemas.
📊
Prometheus + Grafana
Stack de monitoreo para producción. Métricas de sistema y aplicación en tiempo real con dashboards visuales y alertas automáticas.
